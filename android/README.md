# Aegis Android App — Build Guide

This guide explains how to build the Aegis Android app from the existing Next.js web project using Capacitor.

## Prerequisites

1. **Node.js 18+** and **Bun** (for the web app)
2. **Android Studio** (Hedgehog or newer)
3. **JDK 17** (bundled with Android Studio)
4. **Android SDK** (API 34, via Android Studio SDK Manager)

## Step 1: Install Capacitor

```bash
# Install Capacitor dependencies
bun add @capacitor/core @capacitor/cli @capacitor/android

# Initialize Capacitor (if not already done — capacitor.config.json exists)
npx cap init Aegis com.aegis.sms --web-dir=out
```

## Step 2: Build the Web App

The web app must be exported as static HTML for Capacitor to bundle:

```bash
# Add static export to next.config.ts
# (Already configured — the `out` directory is generated by next build)

# Build the web app
bun run build
```

This generates the `out/` directory with all static assets.

## Step 3: Copy Web Assets to Android

```bash
# Copy the built web app into the Android project
npx cap copy android
```

## Step 4: Open in Android Studio

```bash
# Open the Android project in Android Studio
npx cap open android
```

Android Studio will open with the `android/` project loaded.

## Step 5: Build the APK

In Android Studio:

1. Select **Build → Build Bundle(s) / APK(s) → Build APK(s)**
2. Wait for the build to complete
3. The APK will be at `android/app/build/outputs/apk/debug/app-debug.apk`

## Step 6: Test on a Device

1. Enable **Developer Options** on your Android phone (tap Build Number 7 times)
2. Enable **USB Debugging**
3. Connect your phone via USB
4. In Android Studio, click the **Run** button (green play icon)
5. The app will install and launch on your device

## Step 7: Set as Default SMS App

1. Open Aegis on your phone
2. Go to **Settings → Default SMS App**
3. Tap **"Set as Default SMS App"**
4. Android will show the RoleManager dialog — tap **"Set as default"**
5. Aegis is now the default SMS app — incoming SMS will be auto-parsed

## Step 8: Import Existing SMS

After becoming the default SMS app:

1. In **Settings → Default SMS App**, tap **"Import Existing SMS History"**
2. Grant the READ_SMS permission when prompted
3. Aegis will read all existing SMS from your device and parse them

## How It Works

### Incoming SMS Flow (Phase C)

```
1. Android OS receives SMS
   ↓
2. SmsDeliverReceiver.onReceive()
   (SMS_DELIVER action — only default SMS app gets this)
   ↓
3. Extract sender + body from PDUs
   ↓
4. Insert into content://sms provider
   (required — Aegis is now the system of record for SMS)
   ↓
5. AegisSmsBridge.notifySmsReceived(event)
   (queues event for WebView delivery)
   ↓
6. AegisSmsPlugin.deliverSmsEvent(event)
   (Capacitor notifyListeners → JavaScript)
   ↓
7. Web app: window.AegisNative.onSmsReceived callback
   (in src/lib/native-bridge.ts)
   ↓
8. POST /api/inbox/compose
   (runs parseSms → detectScam → categorize)
   ↓
9. If verified transaction → voice pronunciation
   If flagged → security alert
   If OTP → blurred in inbox, gated behind biometric reveal
   ↓
10. AegisNotificationManager.showSmsNotification()
    (system notification with OTP blur + quick-reply)
```

### Bulk Import Flow (Phase C)

```
1. User taps "Import Existing SMS" in Settings
   ↓
2. AegisSmsPlugin.importExistingSms()
   (reads all SMS from content://sms via ContentResolver)
   ↓
3. Returns count to web layer
   ↓
4. Web layer: POST /api/inbox/import with message batch
   ↓
5. Backend: for each SMS → parseSms → detectScam → categorize
   ↓
6. Create SmsMessage + Transaction/FlaggedMessage records
   ↓
7. Return summary: { imported, skipped, transactionsCreated, flaggedCreated }
```

## Permissions

The app requests these permissions (declared in AndroidManifest.xml):

| Permission | Purpose |
|---|---|
| `RECEIVE_SMS` | Receive incoming SMS as default SMS app |
| `READ_SMS` | Read existing SMS history for bulk import |
| `SEND_SMS` | Send quick-reply messages |
| `WRITE_SMS` | Write to the SMS content provider |
| `RECEIVE_MMS` | Receive MMS (required for default SMS app) |
| `RECEIVE_WAP_PUSH` | Receive WAP push (MMS delivery) |
| `INTERNET` | WebView network access |
| `VIBRATE` | Vibrate on incoming SMS |
| `POST_NOTIFICATIONS` | Show SMS notifications (Android 13+) |

## Play Store Compliance

**Important:** Apps with `READ_SMS` permission in the "Default SMS Handler" category must:

1. Be the actual default SMS handler (implement all 4 required components)
2. Provide a demo video showing SMS handling functionality
3. Fill the Restricted Permissions Declaration form in Play Console
4. Have accurate Data Safety form answers

The current app implements all 4 required Android components:
- ✅ `SmsDeliverReceiver` (SMS_DELIVER BroadcastReceiver)
- ✅ `WapPushDeliverReceiver` (WAP_PUSH_DELIVER BroadcastReceiver)
- ✅ `HeadlessSmsSendService` (RESPOND_VIA_MESSAGE service)
- ✅ `MainActivity` with ACTION_SENDTO intent filter (compose Activity)

## Troubleshooting

### "Default SMS app role is not available"
- This happens on emulators without SMS capability
- Test on a real Android device with a SIM card

### "Permission denied reading SMS"
- Ensure `READ_SMS` permission is granted in Settings → Apps → Aegis → Permissions
- The permission is requested at runtime when the user taps "Import"

### WebView shows blank screen
- Run `bun run build` to generate the `out/` directory
- Run `npx cap copy android` to sync web assets
- Check `capacitor.config.json` → `webDir` is set to `"out"`

### SMS not being received
- Verify Aegis is set as the default SMS app (Settings → Default SMS App)
- Check that `SmsDeliverReceiver` is registered in AndroidManifest.xml
- Check logcat for `AegisSmsRx` tag

### OTP not blurred in notifications
- The `AegisNotificationManager` has its own OTP detection
- Check that the message matches one of the OTP patterns in `isOtpMessage()`
